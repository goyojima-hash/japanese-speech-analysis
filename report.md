## 2. モデル別比較

### 2-1. ひらがな出力率・混入率

全モデル・全ファイルで漢字・カタカナ混入は **0%** でした。  
※「ー」（U+30FC 長音符）はひらがな文脈での正規表現として除外

| モデル | 漢字混入率 | カタカナ混入率 | ひらがな出力 |
|--------|-----------|--------------|------------|
| **A (sakasegawa)** | 0% | 0% | ひらがな＋「ー」（長音符） |
| **B (slplab)** | 0% | 0% | 純ひらがな（「ー」なし） |
| **C (vumichien)** | 0% | 0% | ひらがな＋「ー」＋単語間スペース |

**長音符「ー」の扱いについて**

- モデルA・C は長音を「ー」で表現（例: 「にゅーす」）
- モデルB は長音を母音で展開（例: 「にゅうすう」）

J-GRADE の音声採点の観点では、「ー」は発音の伸ばしを正直に表現しており、むしろ自然です。モデルBの母音展開は完全なひらがなですが、原音との乖離が生じる場合があります。

### 2-3. 出力品質（定性評価）

ベトナム語話者音声（vietnamese_japanese_1min.mp3）を参照テキストとして確認：

**モデルA（sakasegawa）出力（抜粋）**
```
んべとともまさいきにっぽんでわとのがていますわたしわいいことともってるわけでわりません
...にゅーすわさいきんおくないですか...はんざいについて...けいさつしょ...
```
→ 「にゅーす」「はんざい」「にっぽん」「べとなむ」など、内容が読み取れる語が多数出現。スペースなしのコンパクトな出力。

**モデルB（slplab）出力（抜粋）**
```
だんうべとたんともしますさいきんにっほんれはぇとのむじんがふふえています...
にゅうすうはさいきおおくないですか...はんざんつい...
```
→ 認識できる語はあるが、「にっほん」「ぇとのむ」など誤認識が散見。長音を母音展開するため語が変化しやすい。

**モデルC（vumichien）出力（抜粋）**
```
だんべっと たんと も し ます さいきん にっぽん で は べとろむ じん が ふえ たい ます が...
はんざ を おこし た べとろむじん の ため に つやく し て い まし た
```
→ 単語間スペース区切り。「つやく」「はんざ（い）」など意味ある語が読み取れるが、処理速度が実用上の問題。

## 3. 話者属性別の傾向

| 話者 | 傾向 |
|------|------|
| **ベトナム人** | 母音・子音の混同が多め。「べとなむ」→「べとのむ」等。モデルAが比較的安定 |
| **ミャンマー人** | 音節境界が明確で、全モデルで比較的読み取りやすい語が出現。「まんご」「かぞく」等が認識可能 |
| **インドネシア人** | 「いもうと」「ゆうじん」「いじめ」等のキーワードが全モデルで概ね認識可能 |

> **注意**: 今回の検証は各話者1ファイル（60秒）のみのため、傾向の断言には追加データが必要。

## 4. 推奨モデル

### J-GRADE 本番環境への実用推奨：**モデルA（sakasegawa）**

**推奨理由：**

1. **処理速度が圧倒的**  
   60秒音声を約10秒で処理（6倍速）。採点ワークフローへの組み込みに支障なし。

2. **ハルシネーションが構造的に発生しない**  
   CTCデコーダ＋言語モデルなしのアーキテクチャのため、発話にない語を「創作」することがない。外国人学習者の不完全な発話でも「聞こえた音だけ」を誠実に出力する。

3. **日本語データ量が圧倒的**  
   ReazonSpeechの35,000時間の事前学習により、B・Cの多言語ベースモデルと比べ日本語音声への適合度が高い。

4. **ひらがなのみ出力を確認済み**  
   漢字・カタカナの混入なし。「ー」（長音符）は発音表現として許容範囲。

5. **ライセンス**：Apache-2.0（商用利用可）

**モデルB（slplab）について：**  
純ひらがな出力（「ー」なし）はチェック観点では優秀だが、長音を母音展開するため語形が変化しやすく、人手での確認コストが増す可能性がある。処理速度はA以下。

**モデルC（vumichien）について：**  
単独動作で60秒音声に5分かかるため、実運用には不向き。単語分かち書きスタイルの出力は用途によっては使いやすいが、速度が致命的。

## 5. 今後の検討事項

- **データ拡充**: 話者1人あたり複数ファイル（難易度・話題・速度の違い）での検証
- **チャンク分割の最適化**: 現在30秒/チャンクだが、文の切れ目を考慮した分割で精度向上の可能性
- **後処理の検討**: モデルA出力を形態素解析して可読性向上（ただし、J-GRADE用途では意味補正なしが前提）
- **モデルAの「ー」安定性**: 公式ドキュメントで「最も不安定なトークン」と注記あり。長母音の多い話者での影響を追加確認推奨
